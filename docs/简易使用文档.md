# Git Save/Load 使用文档

一个 HanaAgent 侧栏 Widget 插件，在聊天界面里管理 Git 仓库。

---

## 快速开始

1. 把插件装进 HanaAgent
2. 在侧栏打开 Git Save/Load 面板
3. 顶部输入你的 Git 仓库路径，回车
4. 开始存档、读档、推送

---

## 功能说明

### 仓库路径

设置插件操作的 Git 仓库。输入完整路径后回车即可。

路径左侧📁按钮会自动填入当前插件绑定的仓库路径（如果已配置）。

### 存档（Commit）

- 输入提交消息，点「一键存档」
- 如果有版本号需求，在左侧输入版本号（如 `1.0.1`），存档时自动打 tag
- 支持 `feat:` / `fix:` / `chore:` 等提交前缀

### 变更文件

显示当前工作区的文件变更列表：

| 标记 | 含义 |
|------|------|
| `??` | 未跟踪的新文件 |
| `M` | 已修改 |
| `A` | 已暂存 |
| `D` | 已删除 |
| `R` | 重命名 |
| `U` | 合并冲突 |

点击文件名查看具体 diff（差异对比）。文件状态旁边的 `+N / -M` 数字显示该文件增删行数。

### 提交记录

- 按时间倒序展示最近提交
- 显示 hash、提交消息、日期
- **点击 hash** 或后面回滚按钮 → 选择回滚模式
- 回滚支持三种模式（见下方说明）

### 回滚

| 模式 | 效果 |
|------|------|
| **soft** | 仅移动 HEAD 指针，代码和暂存区不变 |
| **mixed** | 移动 HEAD + 清空暂存区，代码保留在硬盘 |
| **hard** | 彻底还原到目标版本，未提交的修改全部丢失 |

> hard 模式有风险，点击前确认没有重要未提交内容。

### 暂存（Stash）

临时存放未完成的修改，清空工作区。之后可以恢复（pop）或删除（drop）。

- **暂存当前** — 把当前所有修改暂存起来
- **恢复** — 把选中的暂存内容恢复到工作区
- **删除** — 丢弃选中的暂存

### 推送 / 拉取

- **推送** — 把本地提交推送到远程（默认 origin）
- **拉取** — 从远程拉取最新代码（默认 origin + 当前分支）

设置面板中可调整推送模式和拉取模式：

- 推送模式：`normal` / `force-with-lease` / `force`
- 拉取模式：`merge` / `rebase` / `ff-only`

### GitHub 管理

前提：本机已安装 [GitHub CLI (`gh`)](https://cli.github.com/) 并登录。

| 功能 | 说明 |
|------|------|
| **创建** | 在 GitHub 上建新仓库，自动关联当前本地仓库的 origin |
| **克隆** | 从 GitHub 克隆仓库到本地 |
| **列表** | 查看你的 GitHub 仓库列表（支持指定 owner） |
| **搜索** | 在 GitHub 上搜索公开仓库 |

创建仓库时填写仓库名，选择 Public/Private，可选描述。创建成功后本地仓库自动关联远程。

### 分支管理

- 查看所有本地分支及最新提交
- 创建新分支（可从当前 HEAD 或指定 commit 创建）
- 切换分支
- 删除分支
- 可视化分支画布

#### 分支画布

面板顶部 `🌳` 按钮进入可视化分支视图：

- **方块** = 每个分支
- **连线** = 分支之间的父子继承关系（按命名前缀推断）
- **拖拽** = 自由移动方块位置（布局自动保存）
- **空白处拖拽** = 平移画布
- **右键方块** = 创建分支、切换、设置颜色、添加备注
- **双击连线** = 编辑连线标注文字

### 冲突解决

拉取或合并产生冲突时：

1. 插件自动检测冲突文件
2. 显示冲突文件列表及各冲突块
3. 每个冲突块选择保留「本地的」或「远程的」
4. 应用选择后自动 `git add` 标记为已解决

### 设置

| 配置项 | 说明 |
|--------|------|
| 暂存模式 | 是否包含未跟踪文件（normal / untracked / all） |
| 推送模式 | normal / force-with-lease / force |
| 拉取模式 | merge / rebase / ff-only |
| 默认 diff 模式 | detail（完整差异）/ simple（仅统计） |
| 主题 | light / dark |

---

## 卡片排序

每个功能卡片左上角有 `☰` 把手，拖拽可调整卡片顺序，重启后保留。

---

## 提示

- 修改代码时，插件面板不会自动刷新，需要手动点右上角刷新按钮（🔄）
- 所有数据操作（提交、推送、拉取、回滚）成功后，相关卡片会自动刷新
- 冲突文件请逐个块确认选择，不要跳过
- GitHub 功能依赖 `gh` CLI，请确保已安装并登录
