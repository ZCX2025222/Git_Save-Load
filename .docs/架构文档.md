# Git Save/Load 插件架构文档

## 概述

Git Save/Load 是一个 HanaAgent 平台侧栏 Widget 插件，提供 Git 版本管理的核心功能。插件采用**事件总线 + 卡片组件**架构，每个功能区域为独立卡片，通过事件驱动通信。

- 插件 ID: `git-save-load`
- 版本: 1.0.0
- 平台: HanaAgent (OpenHanako)
- 开发语言: JavaScript (前端) + Node.js (后端)

## 架构

### 事件总线（Event Bus）

插件内部通信的核心机制，实现卡片间的松耦合。

```
Bus.on("commit", fileCard.refresh)    // 文件卡片说：听到commit就刷新
Bus.on("commit", logCard.refresh)     // 日志卡片说：我也要刷新
Bus.emit("commit")                     // 存档按钮说：我提交了
```

**核心 API：**

```javascript
Bus.on(event, callback)     // 订阅事件
Bus.emit(event, data)       // 发射事件
Bus.off(event, callback)    // 取消订阅
```

**事件列表：**

| 事件 | 发射者 | 响应者 |
|------|--------|--------|
| `commit` | 存档按钮 | fileCard, logCard |
| `push` | 推送按钮 | fileCard |
| `pull` | 拉取按钮 | fileCard |
| `reset` | 回滚确认 | fileCard, logCard |
| `branch-switch` | 切换分支 | fileCard, logCard |
| `refresh-all` | 刷新按钮 | 所有已注册卡片 |

### 卡片注册系统

```javascript
Cards.register("fileCard", {
  events: ["commit", "push", "pull", "reset", "branch-switch", "refresh-all"],
  refresh: function() {
    var p = currentPath || getSavedPath();
    if (p) fetch("/api/status?...").then(renderStatus);
  }
});
```

已注册卡片：

| 卡片 ID | 订阅事件 | 功能 |
|---------|----------|------|
| fileCard | commit, push, pull, reset, branch-switch, refresh-all | 变更文件状态 |
| logCard | commit, reset, branch-switch, refresh-all | 提交历史列表 |
| stashCard | commit, stash-push, stash-pop, stash-drop, refresh-all | 暂存列表 |

### 分支画布（独立视图）

分支画布不注册为卡片，而是作为独立的全屏视图，有自己的坐标系、拖拽逻辑和 SVG 连线渲染。

## 实现细节

### 主题系统

使用 CSS 自定义属性实现深色/浅色主题切换：

```css
:root {
  --hana-bg: #f5f6f6;        /* Linear 浅色默认 */
  --hana-accent: #5e6ad2;    /* 蓝紫色主色 */
}
.theme-dark {
  --hana-bg: #010102;         /* Linear 深色 */
}
```

通过 `document.body.classList.toggle("theme-dark")` 切换。主题选择保存在 localStorage + 后端配置。

### 分支颜色

按命名约定自动分配颜色，支持用户自定义覆盖：

```javascript
function branchColor(name) {
  var custom = localStorage.getItem(":color:" + name);
  if (custom) return custom;                    // 自定义颜色优先
  if (name === "master") return "#27a644";      // 绿色
  if (name.startsWith("feature/")) return "#5e6ad2";  // 蓝色
  if (name.startsWith("fix/")) return "#e67e22";      // 橙色
  if (name.startsWith("hotfix/")) return "#e74c3c";   // 红色
  // ...
}
```

### 卡片拖拽排序

使用 HTML5 Drag and Drop API。拖拽时蓝色指示线显示插入位置，松开后保存顺序到 localStorage。

```javascript
container.addEventListener("dragover", function(e) {
  // 检测鼠标在卡片上半部/下半部 → before/after
  // 移动指示线到对应位置
});
container.addEventListener("drop", function(e) {
  // 将源卡片插入到指示线位置
  // 保存新顺序
});
```

### 缓存机制

状态和日志数据缓存 10 秒，写操作时自动清除缓存：

```javascript
function cacheSet(key, data) {
  localStorage.setItem("gsl-cache-" + key, JSON.stringify({ t: Date.now(), d: data }));
}
function cacheGet(key) {
  var raw = JSON.parse(localStorage.getItem("gsl-cache-" + key));
  if (raw && Date.now() - raw.t < 10000) return raw.d;
  return null;
}
```

### 安全措施

- 用户输入经 `escapeHtml()` 转义后入 DOM
- 文件名等数据使用 `data-*` 属性 + 事件委托，避免内联 onclick 注入
- gh 命令参数经 shell escape 后传入
- Git 命令参数双引号包裹
- `windowsHide: true` 避免终端闪现
