# Git Save/Load 插件开发踩坑记录

本文件记录了开发 Git Save/Load 插件过程中遇到的技术问题、踩坑经历和解决方案。

---

## 1. `prompt()` / `confirm()` 在 widget iframe 中被拦截

**现象：** 用户单击📁按钮后完全无反应，因为 Hana 的 widget iframe 禁用了浏览器的 `prompt()` 和 `confirm()`。

**解决方案：** 所有输入改用自定义弹框：

```html
<div id="promptModal">
  <input id="promptInput">
  <button onclick="confirmPrompt()">确定</button>
  <button onclick="cancelPrompt()">取消</button>
</div>
```

全局 `_promptCb` 变量存储回调，`confirmPrompt()` 读取输入值后调用回调。**注意**：`_promptCb` 必须在调用后立即置 `null`，否则嵌套调用时会被覆盖（详见第 9 条）。

## 2. `COMMIT_EDITMSG` 污染 Git 内部状态

**现象：** 每次提交后，用户手动 `git commit` 会弹出上次的旧提交消息。

**根因：** 代码直接写入 `.git/COMMIT_EDITMSG`，这是 Git 每次 commit 自动生成/读取的临时文件。覆盖后会影响后续正常 commit。

**解决方案：** 改用临时文件：

```javascript
const tmpMsg = join(tmpdir(), "git-msg-" + Date.now() + ".txt");
writeFileSync(tmpMsg, message, "utf8");
gitExec(path, `git commit -F "${tmpMsg}"`);
// finally { unlinkSync(tmpMsg); }
```

## 3. `scrollHeight` 在 `overflow:hidden` 时返回 0

**现象：** 搜索面板自动展开后只显示了一点点（约 30px），状态文字看不见。

**根因：** `ensureGhOpen()` 用 `body.scrollHeight` 计算展开高度。但此时所有 tab 内容还是 `display:none`（`switchGhTab` 还未执行），`scrollHeight` 只计算了可见的 tab 按钮高度。

**解决方案：** 先展开到固定值（500px），等 tab 切换完成后再用 `recalcGhBody()` 修正到真实高度：

```javascript
function ensureGhOpen() {
  body.style.maxHeight = "500px";  // 固定值展开
}
function recalcGhBody() {
  body.style.maxHeight = "none";
  void body.offsetHeight;         // 强制回流
  body.style.maxHeight = body.scrollHeight + "px";
}
```

## 4. HTML 标签提早闭合导致内容丢失

**现象：** GitHub 卡片折叠后，创建/克隆/列表/搜索的输入框仍然可见；卡片不能拖拽。

**根因：** ghBody 的 `</div>` 放在了 tab 按钮后面，所有 tab 内容（输入框等）都在 ghBody 外部。浏览器把后面 `panelsArea` 的 `</div>` 当成 ghBody 的闭合。

**解决方案：** 确保 ghBody 的 `</div>` 在所有 tab 内容之后。

**教训：** 修改 HTML 结构后一定要检查所有标签的嵌套关系。建议用一个标签检查工具或手动数开闭标签数量。

## 5. 单引号在 innerHTML onclick 中破坏 JS 语法

**现象：** 分支名中包含 `'` 时（极少见），右键菜单完全失效，整个 JS 报错。

**根因：** `escapeHtml()` 基于 `textContent → innerHTML`，只转义 `<>` 和 `&`，不转义 `'`。分支名拼入 `onclick="switchBranch('${name}')"` 时，`'` 提前关闭了 onclick 字符串。

**解决方案：** 所有动态数据用 `data-*` 属性 + 事件委托，不在内联 onclick 中拼接用户数据：

```javascript
// 坏：onclick 拼接用户数据
`<div onclick="switchBranch('${name}')">`

// 好：data 属性 + 事件委托
`<div data-name="${name}">`
container.onclick = function(e) {
  var name = e.target.getAttribute("data-name");
  if (name) switchBranch(name);
};
```

## 6. `_promptCb` 嵌套调用时被覆盖

**现象：** 基于 commit 创建分支时，第二步（输入分支名）的确认按钮无反应。

**根因：** `confirmPrompt()` 的函数顺序：

```javascript
// 问题代码
function confirmPrompt() {
  var val = input.value;
  cleanupPrompt();
  _promptCb(val);        // ← 内部调用了 customPrompt，设置了新的 _promptCb
  _promptCb = null;       // ← 把刚设置的新回调覆盖了！
}
```

**解决方案：** 先保存再清空：

```javascript
function confirmPrompt() {
  var val = input.value;
  cleanupPrompt();
  var cb = _promptCb;
  _promptCb = null;   // 先清空
  cb(val);            // 再执行
}
```

## 7. `event` 隐式变量在 iframe 中不可靠

**现象：** 搜索框按下回车时，`onkeydown` 事件不触发搜索函数。

**根因：** `onkeydown="if(event.key=='Enter')ghSearch()"` 中的 `event` 是隐式全局变量 `window.event`，在 widget iframe 中可能为 `undefined`。

**解决方案：** 改为显式传参：

```html
<input onkeydown="ghSearchEnter(event)">
```

```javascript
function ghSearchEnter(e) {
  if (e.key === "Enter") ghSearch();
}
```

## 8. `requestAnimationFrame` 在 iframe 中首次调用延迟

**现象：** 第一次搜索时"搜索中..."状态不显示，之后搜索正常。

**根因：** `requestAnimationFrame` 在 iframe 首次加载时可能延迟到数帧后才执行，导致状态文字虽然设置了但没来得及渲染，API 就已经返回了。

**解决方案：** 设置状态文字后等待一帧 + 额外延时再发起请求：

```javascript
st.textContent = "搜索中...";
st.style.display = "";
await new Promise(r => requestAnimationFrame(r));
await new Promise(r => setTimeout(r, 50));  // 额外等待确保渲染
const res = await fetch("/api/gh/search?...");
```

## 9. execSync vs exec 的选择

**问题：** 使用 `exec`（异步）打开外部浏览器时，Windows 会闪现 cmd 窗口。

**解决方案：** 使用 `exec` 时加 `{ windowsHide: true }`：

```javascript
exec(`start "" "${url}"`, { windowsHide: true });
```

`execSync` 默认不显示窗口，但会阻塞主线程。异步操作用 `exec` + `windowsHide`。

## 10. CSS 变量与 Hana 平台冲突

**问题：** 旧版插件使用 `var(--hana-bg, #f5f6f8)` 这类变量，Hana 平台在某些主题下会设置不同的值。

**解决方案：** 在插件 CSS 开头显式覆盖所有 `--hana-*` 变量，确保插件内保持 Linear 设计风格，不受 Hana 主题影响：

```css
:root {
  --hana-bg: #f5f6f6;
  --hana-fg: #000;
  --hana-surface: #fff;
  --hana-border: #e2e5ea;
  --hana-accent: #5e6ad2;
  // ...
}
.theme-dark {
  --hana-bg: #010102;
  // ...
}
```

## 11. `git branch -d` vs `-D`

**问题：** 删除未合并分支时，`git branch -d` 拒绝操作，报错信息让用户困惑。

**解决方案：** 在 UI 中删除分支是主动操作，直接用 `git branch -D` 强制删除。

---

## 12. 模板字面量字符串中丢失闭合 `}`

**问题：** GitHub 创建仓库的 `--description` 参数拼在模板字面量中时，缺少一个 `}` 导致 JS 语法错误。

**根因：** `` `--description "${body.description.replace(/"/g, "'")}` `` 中的 template expression 没有用 `}` 正确闭合。

**教训：** 复杂模板字面量写完后检查所有 `${}` 是否都正确闭合。更安全的方式是把参数拆成数组元素，而不是拼成单一字符串。

---

## 总结

最常踩的坑：

1. **iframe 环境** — 浏览器 API 在 widget iframe 中的行为差异（prompt/confirm/event/requestAnimationFrame）
2. **string concat 注入** — 任何用户数据拼入 HTML/JS 字符串都要考虑转义
3. **DOM 时序** — 先设样式再读布局属性（scrollHeight/offsetHeight）需要强制回流
4. **事件回调覆盖** — 全局回调变量在嵌套调用时要先保存再清空
5. **Git 内部状态** — 不要直接写入 `.git/` 目录下的 Git 内部文件
